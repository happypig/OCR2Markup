Using oXygen XML Author as a component for building a Java application

This project can be used as a starting point for creating an oXygen Author as a Swing component.

Note: To access resources from your CMS you can either use WebDAV (if your server supports it), or you can write a custom protocol handler and plug it into the component using the URL.setURLStreamHandlerFactory method.

Dependencies

  • bundle-frameworks
  • bundle-options
  • bundle-samplefiles

The contents of this project

src The Java sources of the sample application/applet.
pom.xml The Maven project file

Resouces

More details may be found in the oXygen user guide.

Requirements

Specify the license key

Licensing options:

  1. Request an Author Component development license key from us (support@oxygenxml.com). Set this license key (look for a TODO in the Java code) in the AuthorComponentSample Java class.
  2. Use a fixed license key of "evaluation" type - i.e for development only - and put it in the file: "src/main/resources/license.key" (the license key string must be the same as the one received from oXygen, including start/end markers and line breaks)
  3. A floating license key, for the production environment. In this case modify the code to access the specific license server. See the TODOs from AuthorComponentSample.java and Reviewer.java

Licensing informations can be found here

Enabling the logging

You can change the verbosity of the logger by modifiying the src/main/resource/logback.xml

Build and deployment

To build the entire set of files needed by the component, run the following command in the main project directory:

        mvn -pl oxygen-sample-swing-component -am install

Note that the command has been executed in the root project rather that in the module directory. This way, all dependencies of the oxygen-sample-applet module are also built. As a side effect, all the modules in the project will also be built. If you do not need all the modules, you can remove them from their parent's pom.xml file to speed up the build process.

Running the standalone sample applications

You can run the standalone sample using:

      editor.bat
      reviewer.bat

For other platforms than Windows, use directly the Maven command from the bat files.

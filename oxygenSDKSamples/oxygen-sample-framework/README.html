oXygen Sample Framework

This is intended to be a sample startup project for developers that need to extend the oXygen Author editing capabilities for specific XML document types by implementing Java extensions.

Dependencies

  • bundle-frameworks
  • bundle-options
  • bundle-samplefiles

The contents of this project

src The Java source code for the Simple Documentation Framework (SDF). This framework is used as a sample in the Author Developer Guide.
pom.xml The Maven project file
assembly.xml Controls the packaging of the project into a final ZIP containing the framework.
frameworkFiles The set of other resources (CSS files, Schema files, images, etc..) composing the sample framework. These files together with the compiled classes compose the framework deliverable.
doc The java doc.
samples/Oxygen Default Frameworks/src

Sources for the extension implementations Oxygen provides out of the box for Docbook, DITA, TEI and XHTML.

They can be used to alter the corresponding libraries (docbook.jar, dita.jar, etc) in the Oxygen frameworks directory.

This directory is generated on the generate-resources phase of the Maven Build Lifecycle. More information in the How to build the project section.

Resouces

More details and step by step tutorials can be found in the oXygen Author Developer Guide.

How to build the project

The Eclipse IDE may be used to build the project, by running a Maven build. To do this, right-click on the project in Package Explorer and select Run As -> Maven build.... Fill out the Goals field from the Edit Configuration dialog with package and click Run.

This project may also be built in the absence of an Eclipse IDE, using the following command:

mvn package

Note: This will create a ZIP file from the target directory. This package can be unziped into Oxygen installation frameworks directory and Oxygen will automatically load it at startup.

Note: This will also overwrite the samples and doc directories by unpacking artifacts from the SDK.

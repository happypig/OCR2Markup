Samples for the oXygen SDK

The oXygen SDK allows you to extend or create new applications based on the oXygen XML editing technology.

The contents of this project

Main sample sub-projects:
oxygen-sample-swing-component This project is a sample application of using an oXygen XML Author Component which can be embedded in a Java Swing-based application.
oxygen-sample-eclipse-plugin You can use this as a starting project for creating an Eclipse IDE plug-in that extends the oXygen XML Editor plug-in for Eclipse.
oxygen-sample-framework You can use this as a starting project for creating an Java extensions needed by an Author customization for a specific type of XML documents.
oxygen-sample-plugins You can find here a set of plugins for the oXygen XML Editor standalone distribution.

Resources sub-projects:

bundle-frameworks Packs the frameworks you need to deploy to the applet.
bundle-options Packs the default oXygen options you need to deploy to the applet.
bundle-plugins Packs the plugins you need to deploy to the applet. Uses all the plugins from the oxygen-sample-plugins project.
bundle-samplefiles Packs the samples you need to deploy to the applet.

Keep what you need

This sample project is modular. You can keep any of the main sample sub-projects and its dependencies and delete the rest. Make sure you update also the pom.xml file.

For instance if you want to develop an Eclipse plugin you could keep the oxygen-sample-eclipse-plugin and delete from the filesystem the other oxygen-* projects.

How to build the project

All the subprojects can be built directly.

So, running the command
mvn install

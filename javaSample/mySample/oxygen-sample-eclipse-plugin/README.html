Extending the oXygen XML Editor Plugin for Eclipse

This project can be used as a starting point for creating an Eclipse IDE plug-in that extends the oXygen plug-in for Eclipse.

You have samples for adding custom actions to the toolbar and contextual menu, about creating a custom view and accessing the API behind the oXygen Author mode.

This sample Eclipse plugin extends oXygen in the following extension points:

   com.oxygenxml.editor.actionBarContributorCustomizer
   org.eclipse.ui.views

Source code

DILA.eclipse.SampleView Starting from it, you can check which pages are active in the opened XML editors, modify them or add listeners to notify custom views which can provide additional context information. It is possible to access the document model from the Author page directly from your plugin, using most of the Authoring API.
DILA.eclipse.action.CustomActionBarContributorCustomizerImpl Showcases adding actions in the oXygen text page contextual menus, in oXygen toolbar and author's Outline view.

The extension points are referenced in the plugin.xml file.

How to build the project

If you want to use this project from Eclipse, or you need to build against a newer version of Eclipse:

  1. make sure you have installed Oxygen XML Editor plugin in your development Eclipse IDE.
  2. remove the "org.eclipse*" dependencies from the pom.xml file.
  3. convert the project to a plug-in project (in Eclipse, from the project contextual menu: "Configure/Convert to Plug-in Projects"),

Note: This project may be built in the absence of an Eclipse IDE, by use the command:

mvn install

The jar file from the target directory can be copied in the dropins folder of an Eclipse installation.

Note: At build time, the META-INF/MANIFEST.MF file has some of its fields replaced with values from the pom.xml (bundle name, bundle version, bundle vendor). For the rest of the values make sure you keep them in sync in the two files, or adapt the pom.xml to replace others too.
